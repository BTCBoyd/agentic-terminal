# 2026-02-05 - Nostr Threaded Reply Support Added

## Task Completed: NIP-10 Reply Threading

### What Was Built
Updated `post-to-nostr.mjs` to support threaded replies following NIP-10 standard.

### Key Changes
1. **New flags added:**
   - `--reply-to <event_id>` - Links reply to parent event
   - `--mention <pubkey>` - Mentions author for notification

2. **NIP-10 tags automatically added:**
   - `["e", "<event_id>", "", "reply"]` - Marks as reply
   - `["p", "<pubkey>"]` - Mentions author

3. **Backward compatible:**
   - Standalone posts still work without flags
   - Existing scripts unaffected

### Files Created/Modified
1. `post-to-nostr.mjs` - Main script updated
2. `NOSTR-REPLY-GUIDE.md` - Comprehensive guide
3. `NOSTR-REPLY-SUMMARY.md` - Quick reference
4. `test-nostr-reply.mjs` - Validation tests
5. `nostr-decode.mjs` - Helper to convert note1/npub1 to hex

### Usage Examples

**Standalone (unchanged):**
```bash
node post-to-nostr.mjs "Hello Nostr!"
```

**Threaded reply (new):**
```bash
node post-to-nostr.mjs \
  --reply-to <event_id> \
  --mention <author_pubkey> \
  "Great point!"
```

**Decode bech32 to hex:**
```bash
node nostr-decode.mjs note1abc123...
node nostr-decode.mjs npub1xyz789...
```

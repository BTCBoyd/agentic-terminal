# COMMUNICATION TRANSPARENCY TEMPLATE

## When Receiving Large Specs from Boyd

**Template to use:**

```
✅ Spec received and saved to: [filepath]
✅ Task breakdown created: [filepath]
✅ Added to ACTIVE-TASKS.md as [status]

📋 WHAT I UNDERSTAND:
- [Brief summary of key deliverables]

⏱️ ESTIMATED EFFORT:
- [Realistic time estimate]

🚦 NEXT STEPS:
Option A: I can spawn subagent NOW and start implementation (recommend if urgent)
Option B: I can wait for your explicit go-ahead (recommend if you want to review/prioritize first)
Option C: I can create detailed implementation plan for your review before starting

Which would you prefer?
```

## File Naming Convention

**Use these standard names:**

- **Specs from Boyd:** `SPEC-[topic]-[date].md` (e.g., `SPEC-bitcoin-payments-2026-02-06.md`)
- **Tasks derived from specs:** `TASKS-[topic].md` (e.g., `TASKS-bitcoin-payments.md`)
- **Implementation drafts:** `DRAFT-[filename]` (e.g., `DRAFT-pricing-page.html`)
- **Handoff documents:** `HANDOFF-[YYYY-MM-DD].md`
- **Daily memory:** `memory/[YYYY-MM-DD].md`

**Rule:** Never overwrite a spec file. If Boyd sends updated requirements, create a new dated version.

---

**Use this template for ALL large spec deliveries. No exceptions.**
